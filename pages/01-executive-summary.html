1. Executive Summary

Project Overview

The 3D Map Editor MVP is a comprehensive, production-ready application designed for creating and editing 3D architectural environments. This project goes beyond typical MVP approaches by establishing a solid foundation that emphasizes testability, maintainability, and professional-quality code from the very beginning.

Project Goal: Build a functional 3D map editor with modular building components, dual camera modes, and a complete procedural API that enables both manual editing and automated testing.

Critical Design Principles

Two fundamental principles distinguish this project and must be maintained throughout all development phases:

1. Procedural API First

Every user action in the editor must be accessible through a procedural API. This is not optional‚Äîit's a foundational requirement.

Why This Matters

  • Testability: Automated tests can replicate any user action without GUI interaction
  • Scripting: Users can automate repetitive tasks and create custom workflows
  • Debugging: Reproduce bugs programmatically for consistent investigation
  • Batch Operations: Process multiple operations efficiently
  • Remote Control: Enable network-based control and collaboration features

Implementation Approach:

// Example: API-first design
class EditorAPI {
public:
    // Core operations
    Result<EntityID> PlaceComponent(ComponentType type, Vector3 position, Quaternion rotation);
    Result<void> DeleteEntity(EntityID id);
    Result<void> SetEntityTransform(EntityID id, Transform transform);
    
    // Camera operations
    Result<void> SetCameraMode(CameraMode mode);
    Result<void> SetCameraPosition(Vector3 position);
    
    // Scene operations
    Result<void> SaveScene(const std::string& filepath);
    Result<void> LoadScene(const std::string& filepath);
};

// UI code simply calls the API
void OnPlaceWallButton() {
    auto result = editorAPI.PlaceComponent(
        ComponentType::Wall, 
        currentCursorPosition,
        Quaternion::Identity()
    );
    if (!result) {
        ShowError(result.GetError());
    }
}

2. Test-Driven Architecture

The architecture is designed from the ground up to enable automatic testing without human intervention. This goes beyond unit tests to include integration tests, system tests, and even visual validation tests.

Testing Strategy

  • Unit Tests: Test individual components and functions in isolation
  • Integration Tests: Verify interactions between systems
  • System Tests: Test complete workflows end-to-end via the API
  • Regression Tests: Ensure bug fixes stay fixed
  • Performance Tests: Monitor frame rates and resource usage
// Example: Automated test
TEST_F(PlacementSystemTest, PlaceWallWithSnapping) {
    // Setup: Place floor
    auto floor = api.PlaceComponent(ComponentType::Floor, Vector3(0, 0, 0));
    ASSERT_TRUE(floor);
    
    // Action: Place wall
    auto wall = api.PlaceComponent(ComponentType::Wall, Vector3(0, 0, 0));
    ASSERT_TRUE(wall);
    
    // Verify: Wall snaps to floor edge
    auto transform = api.GetEntityTransform(*wall);
    ASSERT_TRUE(transform);
    EXPECT_NEAR(transform->position.y, 0.0f, 0.01f);
    
    // Verify: Collision detection works
    auto hasCollision = api.CheckCollision(*floor, *wall);
    EXPECT_FALSE(hasCollision); // Should not overlap
}

MVP Focus Areas

7 Modular Building Components

The MVP includes seven essential building components, each with distinct behavior and snapping rules:

  1. Walls: Vertical structures that snap to floors and other walls
  2. Floors: Horizontal surfaces that form the base of structures
  3. Stairs: Angled components connecting different elevations
  4. Windows: Transparent openings that snap to walls
  5. Railings: Safety barriers for edges and stairs
  6. Skylights: Roof-mounted transparent components
  7. Roofs: Angled or flat top structures

Dual Camera Modes

Two camera perspectives provide flexibility for different editing tasks:

  • First-Person (FPS): Navigate through the scene as if walking inside
  • Third-Person (TPS): Orbit around the scene for broader perspective

Component Placement and Snapping

Intelligent placement system with:

  • Automatic snapping to nearby components
  • Visual feedback during placement
  • Collision detection and prevention
  • Grid-based alignment options

Basic Lighting System

Essential lighting for scene visualization:

  • Directional light (sun/moon)
  • Point lights for local illumination
  • Basic shadow mapping

Simple Environment

  • Ground plane with grid
  • Basic skybox or gradient sky
  • World origin reference

Essential Editor Tools

  • Select tool: Pick and highlight entities
  • Transform tool: Move, rotate, scale components
  • Delete tool: Remove entities
  • Camera controls: Navigate the scene

Procedural API Foundation

Complete API covering all editor operations, enabling:

  • Programmatic scene creation
  • Automated testing
  • Scripting and batch operations
  • External tool integration

Comprehensive Automated Testing

  • Test suite for all core functionality
  • Continuous integration pipeline
  • Performance benchmarks
  • Regression test coverage

What Is NOT in the MVP

To maintain focus and deliver a quality MVP, the following features are explicitly excluded:

Excluded from MVP:
  • Advanced rendering features (PBR materials, advanced post-processing)
  • Terrain editing and landscape tools
  • Vegetation and foliage systems
  • Advanced physics simulation
  • Multiplayer or networking features
  • AI or pathfinding systems
  • Custom material editor
  • Animation system
  • Particle effects
  • Audio system

Success Criteria

The MVP will be considered successful when:

  1. All 7 component types can be placed, transformed, and deleted
  2. Components snap correctly to adjacent components
  3. Both camera modes function correctly and smoothly
  4. Scenes can be saved to and loaded from disk
  5. Basic lighting illuminates the scene
  6. All functionality is accessible via the procedural API
  7. Automated test suite achieves >80% code coverage
  8. Application runs at 60 FPS on target hardware
  9. No critical bugs in core functionality
  10. Documentation is complete and accurate
